# 骸ノ騎士 - BOSS RUSH

ソウルライク風の「ボス戦のみ」を切り出したブラウザゲーム（プロトタイプ）。
Three.jsによる3D描画・背後視点（ロックオンカメラ）で、巨大なボスと1対1で戦う。
タイトル画面でボスを選択して挑戦する。スマートフォンでの縦持ちプレイを想定。

### 登場ボス（すべてオリジナルキャラクター）

| ボス | 特徴 |
|---|---|
| 骸ノ騎士 グラム | 大剣。振り下ろし・薙ぎ払い・突進・跳躍斬り・回転斬り |
| 雷ノ王 ヴォルガ | 長槍と雷。超高空からの急襲、長距離の突進突き、プレイヤーを追う三連落雷、自分中心の雷光放射。第2形態で槍が雷を纏う |

※ 特定の既存ゲームのキャラクター・名称・音楽等は一切使用していないオリジナル作品です。

## 遊び方

1. `index.html` をブラウザで開く（スマホ推奨・PC可）
2. 「挑戦」をタップして戦闘開始
3. ボスの予備動作（赤い予告範囲）を見て回避し、隙に攻撃する
4. ボスのHPをゼロにすれば討伐完了。討伐時間の自己ベストが記録される

### 操作（スマホ）

| 操作 | 内容 |
|---|---|
| 画面左半分をドラッグ | 移動（ロックオン中はボスの周囲を回り込める） |
| 攻撃（右） | 剣攻撃（スタミナ消費） |
| 回避（右） | 無敵時間つきロール（スティック方向へ。ニュートラルならバックステップ） |
| ガード（左・長押し） | 被ダメージを大幅カット（スタミナを削られる。スタミナ切れでガードクラッシュ） |
| パリイ（左） | 攻撃が当たる直前で押すと無効化し、ボスの体幹を大きく削る |
| 回復（左） | 回復薬を使う（3回まで・使用中は無防備） |

### 操作（PC）

WASD/矢印: 移動 ／ Space or K: 回避 ／ J or Enter: 攻撃 ／ I or Shift(長押し): ガード ／ U or O: パリイ ／ L or H: 回復

## ゲームシステム

- **スタミナ制**: 攻撃・回避・ガードで消費。切れると行動不能になるため攻め時と引き時の管理が必要
- **回避ロール**: 発生直後に無敵時間あり。攻撃をタイミングで「すかす」のが攻略の中心
- **ガード**: 押している間、被ダメージを約85%カット（代わりにスタミナを削られる）。スタミナが尽きるとガードクラッシュして長い硬直
- **パリイ**: ボスの攻撃が当たる瞬間に合わせると完全無効化し、体幹を大きく削る（ハイリスク・ハイリターン）
- **体幹（スタッガー）**: 攻撃・パリイで体幹ゲージが溜まり、崩れた間は与ダメージ増加
- **第2形態**: ボスHP50%以下で行動が高速化し、連撃を使うようになる

## 技術構成

| ファイル | 内容 |
|---|---|
| `index.html` | ゲームエンジン本体（Three.jsによる3D描画） |
| `boss-data.js` | ボス定義データ（HP・AI・攻撃・使用アニメクリップ等） |
| `three.min.js` | Three.js r147（同梱・MIT License）。CDN不要でオフラインでも動作 |
| `GLTFLoader.js` | glTFモデル読み込み用（Three.js付属・MIT License） |
| `models-data.js` | キャラクターモデル（base64埋め込み、約17MB）。file://直開きでもモデルを読み込むための措置 |
| `models/` | 元のglbモデルとライセンス文（KayKit Character Packs, **CC0**） |
| `classic.html` | 旧ワイヤーフレーム版（Canvas 2D自前描画・依存ゼロ）。比較用に保存 |

### クレジット

- キャラクターモデル/アニメーション: [Mixamo](https://www.mixamo.com/)（Adobe）
  - Mixamoの利用規約に基づき、ゲームに組み込んだ形（models-data.js）でのみ使用・公開しています
  - 素材単体の再配布は不可のため、元のFBX/glbファイルはリポジトリに含めていません。モデルを抽出しての二次利用はご遠慮ください
  - 変換: [FBX2glTF](https://github.com/facebookincubator/FBX2glTF)（リポジトリには含まれません）＋ 自作の統合スクリプトでキャラごとに1つのglbへ結合
- 過去バージョンで使用したCC0モデルのライセンス文を models/ に保存: [Quaternius](https://quaternius.com/)、[KayKit Game Assets](https://kaylousberg.com/)
- キャラクターの表示はglTFモデル + 同梱アニメクリップ（攻撃・回避・ガード・詠唱・死亡など）をゲーム状態に応じて再生。武器（大剣・長槍・盾）はコードで生成しモデルの手スロットに装着している
- モデル読み込みに失敗した環境では、従来のプロシージャル生成キャラクター（カプセル体）に自動フォールバックする

- ボスはデータ駆動: `boss-data.js` の配列にオブジェクトを追加すればボスを増やせる（HP・スケール・AI・攻撃のキーフレームまで全てデータ定義。書式はファイル冒頭のコメント参照）
- キャラクターは手続き的な骨格アニメーションの関節位置に、カプセル・装甲メッシュを毎フレーム張り付けて描画（3Dモデルファイル不要）
- 自キャラは右手に剣・左腕に盾。ガード/パリイは盾で受ける
- 効果音は Web Audio API でリアルタイム合成（音源ファイルなし・著作権フリー）
- 記録（討伐回数・ベストタイム）は localStorage に保存
